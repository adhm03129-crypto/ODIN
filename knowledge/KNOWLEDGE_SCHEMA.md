# Knowledge Schema

## Purpose

Defines the common schema used by every knowledge entity within ODIN.

---

# Standard Entity

Every entity should contain:

- ID
- Name
- Description
- Category
- Source
- Version
- Last Updated
- Status

---

# Relationships

Entities should reference other entities instead of duplicating data.

Example:

Hero

↓

Skills

↓

Equipment

↓

Resources

↓

Recommendations

---

# Metadata

Every entity should support metadata including:

- Tags
- Language
- Game Version
- Confidence
- Review Status
