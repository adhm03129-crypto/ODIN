# Knowledge Versioning

## Purpose

Defines how knowledge evolves over time.

Knowledge changes independently from application code.

---

# Version Format

Knowledge follows Semantic Versioning.

MAJOR.MINOR.PATCH

Example:

1.0.0

1.1.0

1.2.5

2.0.0

---

# Change Types

MAJOR

Breaking structural changes.

MINOR

New knowledge.

PATCH

Corrections and updates.

---

# Change Log

Every update should include:

- Version
- Date
- Summary
- Author
